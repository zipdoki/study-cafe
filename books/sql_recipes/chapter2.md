## 4강 데이터

### 1\. 데이터의 종류

업무 데이터: 서비스와 시스템을 운용하기 위한 목적으로 구축된 데이터베이스에 존재하는 데이터

-   트랜잭션 데이터: 구매 데이터, 리뷰 데이터, 게임 플레이 데이터처럼 서비스와 시스템을 통해 사용자의 행동을 기록한 데이터를 나타낸다.
    
-   마스터 데이터: 카테고리 마스터, 상품 마스터처럼 서비스와 시스템이 정의하고 있는 데이터를 마스터 데이터라고 한다.
    

### 2\. 업무 데이터

업무 데이터의 특징

-   데이터의 정밀도가 높다. 데이터의 정합성이 보증된다.
    
-   갱신형 데이터
    
-   다뤄야 하는 테이블의 수가 많다.
    

업무 데이터 축적 방법

-   모든 데이터 변경하기: 날짜를 기반으로 데이터가 계속 누적되는 경우가 아니라면, 데이터 전체를 한꺼번에 바꾸어 최신 상태로 만든다.
    
-   모든 레코드의 스냅샷을 날짜별로 저장하기
    
-   어제와의 변경 사항만 누적하기
    

업무 데이터 다루기

### 3\. 로그 데이터

로그 데이터 축적 방법

1.  태그, SDK를 통해 사용자 장치에서 데이터를 전송하고 출력하기(비컨 형태)
    
    -   비컨(Beacon) 형태: 데이터 Ping 전송
        

```
사용자 행동 발생
    ↓

태그/SDK가 감지
    ↓

서버로 데이터 조용히 전송 (비컨)
    ↓

로그 데이터로 수집
```

2.  서버에서 데이터를 추출하고 출력하기(서버 형태)
